# Grafos

A cada elemento se le llama **nodo** (o vértice), y la relación entre ellos es una **arista**.

## Propiedades de los Grafos

### Conectividad
Un grafo es conectado/conexo si hay un camino entre cualesquiera dos nodos. Los árboles son grafos conectados con caminos únicos. Pueden existir grafos con nodos desconectados; en estos casos cada parte se llama una componente.

### Direccionalidad
Un grafo puede ser o no dirigido, es decir, las aristas solo se pueden recorrer en ciertas direcciones fijadas.

### Ponderabilidad
Existen grafos ponderados o con pesos; la suma de los pesos a lo largo de un camino es el tamaño de ese camino. Es útil para encontrar el camino más corto o más simple.

### Vecinos y Grados
Dos nodos son vecinos o adyacentes si hay una arista/conexión entre ellos. El grado de un nodo es la cantidad de vecinos que tiene. Como cada arista conecta siempre a dos nodos, la suma de grados siempre es: `2 * cantidad_de_aristas`. En grafos dirigidos se habla de grado de entrada y grado de salida.

### Simplicidad
Un grafo es simple si no hay múltiples aristas en un mismo nodo. Generalmente los grafos son simples.

## Representación en Python

### Lista de Adyacencia
Se usa una estructura (normalmente un diccionario de conjuntos) para definir a qué otros nodos está conectado cada nodo.

**Grafo Dirigido**
```python
lst_ady = {
    1: {2},
    2: {3, 4},
    3: {4},
    4: {1}
}
```

**Grafo No Dirigido**
```python
lst_ady = {
    1: {2, 3},
    2: {1, 3},
    3: {1, 2}
}
```

**Grafo Ponderado** (se guarda el nodo junto con el peso)
```python
# Ejemplo: nodo1: {(nodo2, peso2), (nodo3, peso3), ...}
lst_ady = {
    1: {(2, 5)},
    2: {(3, 7), (4, 6)},
    3: {(4, 5)},
    4: {(1, 2)}
}
```

### Matriz de Adyacencia
Solo para grafos pequeños. Se usa una matriz donde la celda `[i][j]` indica el peso (o 1 si no hay peso) de la arista del nodo `i` al nodo `j`.

```python
mtx_ady = [
    [0, 5, 0, 0],
    [0, 0, 7, 6],
    [0, 0, 0, 5],
    [2, 0, 0, 0]
]
```

### Lista de Aristas
Útil cuando se van a procesar todas las aristas. Se almacena cada arista como un conjunto (o tupla) de nodos.

```python
lst_edg = [ {1, 2}, {2, 3}, {2, 4}, {3, 4}, {4, 1} ]
```

**Con pesos** (se puede guardar como tupla con el peso)
```python
lst_edg = [ (1, 2, 5), (2, 3, 7), (2, 4, 6), (3, 4, 5), (4, 1, 2) ]
```

## Recorridos
### DFS
Se basa en backtracking, revisa un camino y vuelve para revisar otro hasta completar el grafo.
No se revisa un mismo nodos 2 veces, para evitarlo usa una pila.
```python
def dfs(grafo, nodo, visitados):
    if nodo not in visitados:       # DFS no repite nodos
        visitados.append(nodo)     # Se guarda como ya visitado

        print(nodo)     # O cualquier otra operación

        for vecino in grafo[nodo]:      # Se repite el proceso en los vecinos
            dfs(grafo, vecino, visitados)
```

### BFS
La revisión de los nodos es por nivel, va creciendo alrededor del nodo de incio hasta completar el grafo.
Utiliza una cola para saber que nodos revisar y se deben guardar los visitados para no repetir
```python
def bfs(grafo, nodo):
    visitados = []      # BFS no repite nodos
    cola = [nodo]       # Una cola para ir avanzando

    while cola:     # Repetir hasta que se revisen todos los nodos
        nodo = cola.pop(0)      # Se extrae el nodo a revisar de la cola

        if nodo not in visitados:
            visitados.append(nodo)      # Se guarda como visitado

            print(nodo)     # O cualquier otra operación

            for vecino in grafo[nodo]:      # Se guardan los vecinos en la cola
                cola.append(vecino)
```
